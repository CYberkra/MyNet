# 历史复盘与运行中结果体验说明

版本：v0.8.0-alpha.35

## 目标

本轮围绕“导入数据集骨架后一键运行，并能持续看到历史、正在运行、即将运行的模型”继续优化体验。

用户后续应按以下主线工作：

```text
设计数据集骨架
  → 导入 manifest
  → 合同检查 / 路径迁移
  → 预检任务
  → 一键运行
  → 实时查看 B-scan
  → 历史页复盘 / 定位失败 / 只重跑 failed
```

## 已完成的体验优化

### 1. 历史页右键菜单

在“历史与结果”页的树状视图或历史卡片上右键，可执行：

- 打开 case 文件夹；
- 打开 QC JSON；
- 复制失败原因；
- 复制路径摘要；
- 显示失败定位；
- 只重跑 failed。

这避免用户反复在资源管理器里手动定位 `models/case_xxxxxx`、`outputs`、`bscan_qc_report.json`。

### 2. 历史页 pending / running / done / failed 统一展示

历史页不再只显示完成记录。导入骨架后，未运行记录也可进入树状视图，便于统一查看：

- 即将运行的模型；
- 正在运行的模型；
- 已完成结果；
- 失败待处理结果。

### 3. 批量页最近 B-scan 对比条

批量运行时，当前完成的 B-scan 会进入“最近完成的 B-scan”对比条，最多保留 5 个 variant。这样可以在运行期间快速观察：

- raw；
- target_only；
- background_only；
- clutter_only；
- air_only；
- clutter_gt 派生结果。

### 4. 失败定位更结构化

历史详情区会显示：

- manifest；
- case_dir；
- bscan 文件；
- QC JSON；
- status；
- reason。

右键复制路径摘要时会复制 JSON，方便直接发给开发者或新会话审计。

## 设计约束

- 历史页不引入数据库，仍基于 manifest / QC / marker 文件构建视图。
- 路径打开只针对当前记录，不修改 workspace。
- 路径迁移仍由“迁移/修复路径”工具负责，历史页只做定位和复盘。
- 批量页最近 B-scan 只保留轻量内存预览，不写入额外持久文件。

## 后续可继续优化

1. 历史页增加“失败原因聚合”面板，按错误类型统计失败 case。
2. 批量页增加预计剩余时间估算。
3. 历史页增加“按 family / variant / status 快速筛选”。
4. 运行看板增加 GPU runtime profile 与当前机器名称。
