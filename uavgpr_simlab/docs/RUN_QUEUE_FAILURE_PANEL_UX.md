# 运行队列与失败聚合体验说明（v0.8.0-alpha.36）

本轮继续强化“导入数据集骨架 → 一键运行 → 实时监控 → 历史复盘”的主流程。

## 新增能力

1. 批量页新增运行队列树：按 case / variant 展示等待、运行中、完成、失败。
2. 批量页新增失败原因聚合面板：把 failed 任务按首行错误摘要聚合，便于判断是环境问题、模型问题还是后处理问题。
3. 队列树支持双击跳转历史页：已有历史 marker 的任务会自动选中对应记录；pending 任务会提示尚未产生历史记录。
4. 相关逻辑拆入 `gui/controllers/batch_queue_panel.py`，保持批量 controller 边界清晰。

## 操作建议

导入新骨架后，先看队列树确认任务规模；运行过程中看 running 和最近完成 B-scan；失败后先看失败聚合面板，再使用 failed-only 重跑。
