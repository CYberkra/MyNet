# UAVGPR_APPLICATION_CONTEXT_FROM_PDF - v0.7.23

## 来源

用户上传的 `UavGPR to 蒋总.pdf`。该 PDF 用于业务背景理解，不作为源码审计证据，也不直接证明当前软件功能已经全部实现。

## 对软件需求有参考价值的信息

1. 课题对象是低频无人机航空探地雷达装备及配套软件，测试目标包括 20 m 以浅基覆界面探测和钻孔资料验证。
2. 软件侧指标涉及文件/数据管理、数据可视化、多种数据处理、偏移成像与解译等能力。
3. 现场数据流程是：数据获取 → 数据处理 → 数据成像 → 成果整理。
4. 数据处理链路包括低频漂移校正、零时校正、吸收衰减补偿、背景杂波抑制、尖锐杂波抑制，并可进一步使用 SVD、小波等杂波/噪声抑制方法。
5. 成像与解释链路包括逆时偏移、基岩界面识别和全波形反演等更高级能力。
6. 示例现场采集参数包括 20-170 MHz 频段、501 个频点、0-700 ns 观测时窗、约 10 道/s 采集频率等；这些信息对真实 CSV 解析、B-scan 时间窗、QC 文档和未来实测数据导入模板有参考价值。

## 本轮处理原则

- 不把 PDF 的参数硬编码进仿真默认值；
- 不把 PDF 的验收结论当成当前软件已实现证明；
- 只把它作为后续真实数据接入、用户手册和验收模板的业务背景材料。

## 建议转化为后续任务

1. 在用户手册中增加“实测 UAV-GPR 数据流”章节。
2. 在真实 CSV 导入页增加可选 metadata 面板，例如频点数、时窗、采集频率、飞行高度、测线编号。
3. 在 QC 报告中保留 SNR、时间窗、道数、处理方法等字段，以便与测试报告口径对齐。
4. 未来如果接入成像/解释模块，应把 RTM、Mask-RCNN、FWI 作为独立高级模块，不应塞入当前批量仿真页。
