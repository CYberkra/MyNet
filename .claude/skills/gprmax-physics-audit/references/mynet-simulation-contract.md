# MyNet / PGDA Simulation Contract

## Formal Data Boundary

- Line9 is strict test-only.
- LineX1 is review-only/excluded.
- Do not derive formal simulation geometry, label curves, time distributions, terrain templates, or promotion thresholds from Line9.
- Line9-conditioned simulations are development-only even if their electromagnetic fields were generated by gprMax.

## Current Acquisition Target

- 2D x-y model with one z cell and z-polarized line source.
- Canonical output: 501 samples over 0-700 ns.
- Long-line diagnostic target: 128 traces, approximately 1.7 m spacing, approximately 216 m scan span.
- Current provisional source: Ricker wavelet in the 50-65 MHz range, 0.2 m Tx/Rx separation. Choose the exact frequency together with the grid using the solver's dispersion report.
- Domain margins must be much larger than the scan span and must include PML plus physical guards.

These values remain provisional until matched to verified hardware metadata. Record any override in the manifest.

## Source-Waveform Policy

- Keep the 55 MHz Ricker wavelet as the controlled baseline while geometry and materials are being revised. Ricker is the normalized second derivative of a Gaussian in the reviewed gprMax 3.1.7 implementation, not a plain Gaussian pulse.
- Do not prefer the plain Gaussian for the ideal free-radiating Hertzian line source merely because its time trace looks smoother. It is unipolar and DC-rich, so it is a poor default for the AC-coupled radiated pulse being approximated here.
- A normalized first-derivative Gaussian is a valid bipolar, near-zero-DC ablation, but changing it also changes phase and bandwidth. Compare it only in a dedicated source ablation with geometry, materials, grid, gain, and pair-control files held fixed.
- The preferred long-term source is a measured, dewow-corrected system pulse supplied through `#excitation_file`, once acquisition hardware and a representative direct-wave/air-reference waveform are traceably available. Preserve the raw measured pulse, preprocessing recipe, sample interval, normalization, and hash.
- Never tune source type against Line9 test-label accuracy. Source promotion must use independent calibration evidence plus paired-control detectability and morphology checks.

## Long-Line Domain Sizing

- Size the lateral domain from the protected signal window, not from a visually convenient scan-to-domain ratio. For an airborne source, a conservative free-space side-boundary round trip is `t_boundary = 2 * guard / c`, where `guard` is measured from the scan endpoint to the inner PML face.
- To keep every possible lateral return outside the full 700 ns solver window, use about 110 m of physical guard per side. With the current 215.9 m scan, 0.2 m Tx/Rx offset, and 3 m PML, this gives a 442.1 m domain.
- For an overnight diagnostic whose protected target/search interval ends near 500 ns, an 80 m physical guard per side gives a 533.7 ns earliest free-space round trip. The resulting 382.1 m domain is an allowed pilot optimization, but every family must still pass late-time and endpoint audits before promotion.
- Do not reduce below 75 m physical side guard for the current 320-500 ns target/search interval. A roughly 350 m domain would place the fastest lateral return inside the target band and is not accepted merely because PML is present.
- Keep the 60-cell (3 m) PML in addition to the physical guard. Record left/right physical guards, boundary round-trip time, latest geometric reference, and clearance in every manifest.
- Treat the first 382.1 m case as an exact cropped/shifted domain-equivalence control of an accepted larger-domain geometry. Compare signed full/control target response, endpoint energy, and late energy before using the smaller domain for formal families.

Runtime evidence on the reviewed RTX 5070 setup: MACRO04 at 480.1 x 45 m completed 128 full traces in about 29.7 minutes and 128 control traces in about 31.3 minutes. Since the 2D cell count scales linearly with domain width, 382.1 m is expected to reduce solver cost by about 20.4%, bringing ten two-sided pairs close to an eight-hour overnight batch rather than roughly ten hours.

## Strict Pair Contract

For every positive case, create at minimum:

- `full_scene`: all geometry and basal contrast;
- `no_basal_contrast`: same stochastic realization and acquisition with basal contrast removed;
- optional `air_reference`: useful for direct-wave diagnostics, not required for initial pair acceptance.

Full and control must share the same geometry-index HDF5 and the same upper-material mapping. Only transition/basal material properties may differ. Hash all files.

## Geometry Goals

The real Line9 target is not a uniformly dominant reflector. Simulations should support:

- a continuous but multiscale basal surface;
- finite weathered transition thickness;
- slowly varying cover properties and attenuation;
- locally variable target strength and occasional dropout;
- coherent non-target background events without vertical partition walls;
- independent stochastic families and deterministic seeds.

Do not copy Line9 geometry to obtain visual similarity. Match broad physical statistics only after the formal split is locked.

## Acceptance Gates

Before a case may train a formal model:

1. source and geometry provenance are complete;
2. no Line9 conditioning is present;
3. geometry-only and static audit pass;
4. full/control pair alignment passes exactly;
5. visible-phase extraction is validated against the signed pair response;
6. target is neither invisible nor trivially dominant;
7. no boundary, vertical-wall, or endpoint artifact explains the label;
8. an explicit human decision and hashes are recorded.

Case count is not independent sample count. Split by `scene_family_id`, not exported windows.

For label generation, distinguish three objects explicitly:

- geometric/reference arrival: ray/path estimate before waveform phase;
- matched-contrast envelope centre: continuous target-wavelet support;
- visible signed phase: a consistent positive or negative lobe selected inside that continuous support.

Do not save the envelope-centre path under a visible-phase filename. The project extractor uses a two-stage path: a continuous Hilbert-envelope path rejects unrelated events, then a continuous absolute signed-response path selects the actual lobe within the configured phase window.

## Comparison Protocol

Compare simulations, Pilot001, and real data only with identical:

- trace span and sample grid;
- time crop;
- gain function;
- normalization and color limits;
- acquisition/profile orientation;
- distance-axis definition.

Report target/background RMS, target continuity, amplitude CV/dropout, interface range/curvature, and late-boundary energy. Do not promote from a single attractive preview.

Use one shared physical color scale for a strict full/control pair. Across simulation, legacy Pilot data, and measured lines whose amplitude units are not calibrated to each other, use the same robust scaling algorithm independently per dataset for morphology only, disclose every scale factor, and never compare brightness across rows.
